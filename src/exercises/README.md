# Exercises

Practice exercises for each module. Try to solve these before looking at solutions.

## Module 2: Basic Types

### Exercise 1: Type Annotations
Create a function that takes a name (string) and age (number) and returns a formatted string.

### Exercise 2: Union Types
Create a function that accepts either a string or number and returns the length (for strings) or the number itself (for numbers).

### Exercise 3: Type Guards
Create a function that takes `string | number | boolean` and returns a string representation with type information.

## Module 3: Advanced Types

### Exercise 1: Generic Stack
Implement a generic Stack class with `push`, `pop`, `peek`, and `isEmpty` methods.

### Exercise 2: Utility Types
Create a type that makes all properties of a User interface optional except `id`.

### Exercise 3: Conditional Types
Create a type that extracts the return type of a function.

## Module 4: Classes

### Exercise 1: Inheritance
Create a `Vehicle` base class and `Car` and `Motorcycle` subclasses.

### Exercise 2: Abstract Classes
Create an abstract `Shape` class with abstract `getArea()` method, then implement `Circle` and `Rectangle`.

## Module 6: Design Patterns

### Exercise 1: Singleton
Implement a Configuration class that can only have one instance.

### Exercise 2: Factory
Create a factory for creating different types of notifications (Email, SMS, Push).

### Exercise 3: Observer
Implement a simple event system where listeners can subscribe to events.

## Module 7: Web Development

### Exercise 1: DOM Manipulation
Create a function that safely gets an element by ID and adds a click handler.

### Exercise 2: API Client
Create a typed API client for fetching user data from an API.

## Module 11: Backend

### Exercise 1: Express Routes
Create Express routes for CRUD operations on a Todo model.

### Exercise 2: Validation
Create validation middleware for user registration.

**Tip:** Create your solutions in separate files and compare with reference implementations.
